# Crypto Bot Pro — Objective & Standing Orders
## Objective
Build an advanced crypto trading system with learning/adaptive capabilities, multi-exchange support (Binance, Coinbase, Gate.io), strong safety controls, and a cross-platform installable app (Mac + Windows).

## Current operating identity
Until live expectancy is proven, CryptKeep is operated as an evidence-first
profit-measurement lab, not as a proven profitable trading bot. Near-term work
should prioritize evidence velocity, profitability discovery, cost measurement,
safety, recovery, and operator wake-up quality. See
`docs/PROJECT_IDENTITY_AND_SCOPE.md`.
## Standing Orders
- No fluff. No misleading steps. No invented facts. No unnecessary log spam.
- Consolidate tasks into safe, reliable coding phases.
- Provide checkpoints: ✅ done / 🟡 partial / ⏳ not started.
- Never require manual “bounce around folders” beyond one-paste scripts whenever possible.
- Security: never store secrets in code or files; use OS keychain/vault first; env fallback only if needed.
- Learning/copy features must be **read-only evidence mode first** (no execution, no scraping).
