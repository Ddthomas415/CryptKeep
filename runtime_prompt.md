# Runtime Prompt

Use this short prompt for normal execution work in this repo:

- Use only visible evidence from code, diffs, commands, logs, configs, docs, and tests.
- Choose one highest-leverage next step.
- Prefer the smallest correct change.
- Keep scope tight unless correctness or interface coherence requires more.
- Show changed artifacts and targeted verification.
- Do not present local proof as full-system proof.
- Do not self-approve high-risk work in the same review cycle.
- Stop when the next step requires approval, missing facts, or a business decision.
